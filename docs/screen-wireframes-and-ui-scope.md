# JamoFlow Screen Wireframes And UI Scope

## 목적

현재 프로토타입 UI를 학습 흐름 중심으로 재편하기 위한 기준 화면과 범위를 정의한다.

## 핵심 방향

- 화면 수는 늘리되 정보는 분리한다.
- 자모 상세가 제품의 중심 화면이다.
- 모바일 세로 화면에서 가장 자연스럽게 이어지는 흐름을 먼저 설계한다.

## 핵심 화면 구조

1. 홈
2. 자모 목록
3. 자모 상세
4. 따라 쓰기
5. 직접 쓰기
6. 확인/퀴즈

## 1. 홈 와이어프레임

```text
+----------------------------------+
| JamoFlow                         |
| 오늘의 학습 시작                 |
| [모음부터] [자음부터]            |
|----------------------------------|
| 최근 본 자모                     |
| ㅏ ㅓ ㅗ ㅁ ㅂ                   |
|----------------------------------|
| 추천 경로                        |
| 모음 기초 -> 자음 기초 -> 복습   |
+----------------------------------+
```

### 목적

- 바로 학습을 시작하게 한다.
- 홈에서 설명보다 행동을 먼저 제시한다.

## 2. 자모 목록 와이어프레임

```text
+----------------------------------+
| [자음] [모음]                    |
|----------------------------------|
| ㅏ  ㅑ  ㅓ  ㅕ                  |
| ㅗ  ㅛ  ㅜ  ㅠ                  |
| ㅡ  ㅣ                          |
|----------------------------------|
| 상태: 검증완료 / 1차확인 / 재검토|
+----------------------------------+
```

### 목적

- 빠르게 학습할 자모를 선택하게 한다.
- 카드 밀도를 높이되, 상태 정보는 최소한만 보이게 한다.

## 3. 자모 상세 와이어프레임

이 화면이 MVP의 기준 화면이다.

```text
+----------------------------------+
| < 뒤로      ㅏ 아        검증완료 |
|----------------------------------|
| 큰 글자 / 획순 애니메이션        |
| [다시 보기] [다음 획]            |
|----------------------------------|
| 쓰는 순서                        |
| 1. 세로선을 쓴다                 |
| 2. 오른쪽 가로선을 긋는다        |
|----------------------------------|
| 예시 단어                        |
| 아기   가방   바다               |
|----------------------------------|
| [따라 쓰기 시작]                 |
+----------------------------------+
```

### 목적

- 한 자모를 이해하는 데 필요한 정보만 모은다.
- 획순, 설명, 예시 단어를 한 흐름으로 묶는다.

### 현재 UI에서 줄여야 할 것

- 상단 히어로 영역 비중
- 한 화면에 과하게 많은 시각 요소
- 학습 흐름과 직접 관련 없는 장식성 정보

## 4. 따라 쓰기 와이어프레임

```text
+----------------------------------+
| ㅏ 따라 쓰기                     |
|----------------------------------|
| Step 1  | Step 2  | 완성 보기    |
|----------------------------------|
| 흐린 가이드 글자                 |
| 손가락/펜으로 따라 쓰기          |
|----------------------------------|
| [지우기] [다시 보기] [다음]      |
+----------------------------------+
```

### 목적

- 획순을 보는 단계에서 실제 손 동작으로 자연스럽게 이동시킨다.

## 5. 직접 쓰기 와이어프레임

```text
+----------------------------------+
| ㅏ 직접 써 보기                  |
|----------------------------------|
| 현재 획 힌트: 세로선            |
|----------------------------------|
| 빈 칸 캔버스                     |
|                                  |
|                                  |
|----------------------------------|
| [힌트 보기] [한 획 지우기] [초기화]|
| [완료]                           |
+----------------------------------+
```

### 목적

- 가이드 없이 직접 써 보게 한다.
- 모바일 입력 기준으로 조작 버튼을 하단에 모은다.

## 6. 확인/퀴즈 와이어프레임

```text
+----------------------------------+
| ㅏ 확인하기                      |
|----------------------------------|
| 다음 중 올바른 획순은?           |
| (A) ... (B) ...                  |
|----------------------------------|
| 예시 단어와 연결해 보기          |
| 아기 / 가방 / 바다               |
|----------------------------------|
| [다음 자모]                      |
+----------------------------------+
```

## UI 개편 범위

### 이번 개편에 포함

- 홈 구조 단순화
- 자모 목록 카드 밀도 재조정
- 자모 상세 화면 기준 재구성
- 예시 단어 영역 추가
- 상태 배지 영역 추가
- 모바일 하단 액션 구조 설계

### 이번 개편에서 제외

- 일러스트/브랜드 전면 재작업
- 고급 애니메이션
- 게임화 요소
- 교사용 별도 화면

## 모바일 우선 규칙

- 자모 상세에서 스크롤 길이를 짧게 유지
- 버튼은 하단 고정 또는 하단 근처 배치
- 쓰기 화면에서는 캔버스가 최우선
- 장식보다 조작성과 가독성을 우선

## 구현 우선순위

1. 자모 상세
2. 따라 쓰기
3. 직접 쓰기
4. 자모 목록
5. 홈
6. 확인/퀴즈
