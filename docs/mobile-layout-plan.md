# JamoFlow Mobile Layout Plan

## 목적

현재 상세 중심 UI와 직접 쓰기 프로토타입을
모바일 세로 화면 기준으로 안정적으로 정리하기 위한 계획 문서다.

## 적용 범위

- 홈
- 자모 목록
- 자모 상세
- 따라 쓰기
- 직접 쓰기

## 핵심 원칙

1. 상세 화면에서 첫 화면 내 핵심 정보가 보여야 한다.
2. 직접 쓰기 캔버스는 모바일에서 가장 큰 영역을 차지해야 한다.
3. 버튼은 손가락 기준으로 누르기 쉬워야 한다.
4. 스크롤 길이는 짧고, 단계 전환은 명확해야 한다.

## 현재 문제

- 상단 히어로 영역이 아직 모바일에서 길다.
- 자모 상세, 예시 단어, 워크북, 직접 쓰기가 한 페이지에 길게 쌓인다.
- 직접 쓰기 컨트롤은 들어갔지만 단계 전환 CTA가 아직 약하다.
- 홈 화면은 아직 제품형 모바일 진입 화면으로 분리되지 않았다.

## 화면별 정리안

### 1. 홈

- 상단 소개 카피 축소
- `모음부터 시작`, `자음부터 시작` CTA를 첫 뷰포트 안에 배치
- 최근 자모/추천 경로는 접거나 축약

### 2. 자모 목록

- 자모 카드 높이 축소
- 상태 라벨은 짧은 칩 유지
- 한 화면에 더 많은 자모를 보이게 조정

### 3. 자모 상세

- 현재 구조 유지
- 상세 메타 3칸은 모바일에서 1열 유지
- 검증 메모는 짧게 유지
- 예시 단어 카드는 1열 스택 유지

### 4. 따라 쓰기

- 모바일에서는 preview 칸 수를 줄이거나 접기 가능 구조 고려
- 단계 설명과 실제 쓰기 시작 CTA를 더 가깝게 배치

### 5. 직접 쓰기

- 캔버스 높이 최우선
- 상태 박스는 1열
- 컨트롤 버튼은 하단 2행까지 허용
- 다음 단계 CTA를 컨트롤 아래 추가 검토

## 우선 구현 항목

1. [x] 히어로 높이 축소
2. [x] 홈 CTA 우선 배치
3. [ ] 자모 목록 카드 밀도 증가
4. [x] 직접 쓰기 CTA 구조 추가

## 완료 기준

- 모바일 세로 화면에서 첫 학습 진입이 쉬움
- 자모 상세의 핵심 정보가 과도하게 길지 않음
- 직접 쓰기 영역이 충분히 큼
- 하단 버튼이 손가락 기준으로 누르기 쉬움

## 현재 반영 상태

- 히어로 소개 밀도 축소
- 홈 CTA 추가
- 상세 -> 따라 쓰기 -> 직접 쓰기 CTA 연결
- 직접 쓰기 영역 높이 보강
- 자모 목록 카드 밀도 증가는 후속 과제
