# JamoFlow Mobile Writing Requirements

## 목적

모바일 브라우저에서 한글 자모를 손가락 또는 펜으로 따라 쓸 수 있는 직접 쓰기 인터랙션의 요구사항을 정리한다.

## 왜 모바일 기준이 먼저 필요한가

- 한글 쓰기 학습은 태블릿/휴대폰에서 사용할 가능성이 높다.
- 데스크톱 마우스 입력만 기준으로 설계하면 실제 학습감이 떨어진다.
- 손가락 입력은 정확도보다 흐름과 피드백 설계가 더 중요하다.

## 입력 요구사항

- `pointer events`를 기본 입력 레이어로 사용
- 터치, 펜, 마우스를 같은 이벤트 모델로 수용
- 캔버스 내부에서는 페이지 스크롤보다 쓰기 입력이 우선
- 입력 중 끊김이 적고 저사양 모바일에서도 유지되어야 함

## UI 요구사항

- 세로 화면 기준 캔버스 우선 배치
- 손가락이 가려도 볼 수 있게 현재 획 힌트는 캔버스 바깥에도 표시
- 버튼은 한 손 조작 가능한 크기 유지
- `다시 쓰기`, `지우기`, `힌트 보기`, `다음` 버튼을 하단 고정 영역에 둘 것

## 학습 요구사항

- 현재 획만 강조하는 모드 필요
- 전체 획순 다시 보기 필요
- 가이드 글자 위 따라 쓰기와 빈 칸 직접 쓰기 모드 분리
- MVP에서는 완벽한 필기 인식보다 반복 경험을 우선

## 기술 요구사항

- 벡터 기반 또는 경량 캔버스 드로잉
- 획 단위 undo 지원
- 디바이스 픽셀 비율 대응
- 입력 좌표와 화면 스케일 분리

## 리스크

- 손가락이 캔버스를 가리는 문제
- 스크롤과 드로잉 충돌
- 작은 화면에서 버튼과 캔버스가 함께 답답해지는 문제
- 저사양 기기에서 잔상 또는 지연 발생 가능성

## MVP 판정 범위

- `입력 가능`
- `지우기 가능`
- `다시 쓰기 가능`
- `획순 힌트 토글 가능`

정교한 정오 판정은 MVP 이후 단계로 미룬다.
