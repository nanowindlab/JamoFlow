# JamoFlow Writing Expansion Plan

## 목적

현재 직접 쓰기 프로토타입을 다음 단계 기능으로 확장하기 위한 계획 문서다.

## 현재 상태

- `pointer events` 기반 입력 가능
- 힌트 토글 가능
- 한 획 지우기 가능
- 전체 다시 쓰기 가능
- 자모 변경 시 상태 초기화됨

## 다음 확장 흐름

```text
기본 프로토타입
  -> 단계 전환 연결
  -> 모바일 액션 강화
  -> 결과 피드백 추가
  -> 학습 완료 흐름 연결
```

## 확장 단계

### 단계 1. 학습 흐름 연결

- [x] 직접 쓰기 위에 현재 단계 표시 강화
- [x] 직접 쓰기 아래에 `다음 자모`, `처음부터 다시` CTA 추가
- [x] 상세 -> 따라 쓰기 -> 직접 쓰기 흐름 연결

### 단계 2. 모바일 액션 강화

- [x] 하단 버튼 레이아웃 고정
- 버튼 우선순위 정리
- 캔버스 높이와 여백 재조정

### 단계 3. 기본 피드백 추가

- [x] 획 수 기준 진행 상태 표시
- [x] 모든 기준 획 힌트 소진 시 완료 메시지 노출
- [x] 완성 후 다음 행동 선택지 제공

### 단계 4. 향후 후보

- 가이드 위 따라 쓰기 모드
- [x] 아주 단순한 완료 판정 보조 규칙 검토
- 저장 없이도 세션 내 반복 추천

## 제외 범위

- 정밀 필기 인식
- 서버 저장
- 점수 기반 게이미피케이션

## 현재 후속 우선순위

1. [x] 직접 쓰기 CTA 연결
2. [x] 완료 후 행동 유도
3. [x] 모바일 버튼 구조 안정화
4. [x] 최소 피드백 추가
5. [x] 아주 단순한 완료 판정 보조 규칙 검토

## 현재 보조 규칙

- 기준 획 수 충족
- 각 입력 획의 최소 점 수 충족
- 전체 이동 길이 최소값 충족

이 규칙은 `정답 판정`이 아니라
`너무 짧은 입력을 알려 주는 non-blocking assist`이다.
