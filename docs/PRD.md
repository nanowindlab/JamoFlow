# JamoFlow PRD

## 1. 제품 개요

JamoFlow는 한국어 입문 학습자가 자음과 모음의 정확한 획순을 보고, 따라 쓰고, 직접 써 보며 익히는 한글 쓰기 학습 앱이다.

현재 프로토타입은 기본 자음 14개와 기본 모음 10개의 획순 시각화에 초점을 두고 있다. 다음 단계의 핵심은 단순 데모를 넘어, 모바일에서도 자연스럽게 사용할 수 있는 실제 학습 흐름으로 발전시키는 것이다.

## 2. 문제 정의

- 기존 한글 쓰기 자료는 정적 워크북 중심이라 반복 연습과 즉각 피드백이 약하다.
- 현재 프로젝트의 획순 데이터는 일부 자모에서 정확성 검증이 아직 끝나지 않았다.
- 예시 단어, 학습 흐름, 인터랙션, 모바일 사용성이 아직 제품 수준으로 정리되지 않았다.

## 3. 목표

### 제품 목표

- 학습자가 자모의 올바른 획순을 신뢰할 수 있게 제공한다.
- 자모별 예시 단어까지 포함해 기억과 발음을 같이 연결한다.
- `보기 → 따라 쓰기 → 직접 쓰기 → 확인`의 학습 흐름을 완성한다.
- 모바일 브라우저에서 손가락으로 쓰기 연습이 가능한 구조를 만든다.

### 사용자 목표

- 자음과 모음의 올바른 쓰는 순서를 빠르게 이해한다.
- 보고 따라 쓰는 단계에서 부담 없이 반복 학습한다.
- 단어 예시를 통해 글자와 실제 사용을 함께 익힌다.

## 4. 비목표

- 로그인/회원 시스템
- 서버 기반 학습 기록 저장
- AI 필기 인식 채점
- 교사용 관리 대시보드
- 문장/문단 수준 쓰기 학습

## 5. 핵심 사용자

- 한국어 입문 학습자
- 다문화/이주 배경 초급 학습자
- 유아/초등 한글 입문자
- 보호자 또는 교사와 함께 쓰기 학습을 하는 사용자

## 6. 핵심 원칙

- 정확성 우선: 획순은 검증되지 않으면 확정으로 취급하지 않는다.
- 학습 흐름 우선: 단순 시각화보다 실제 학습 경험을 중심에 둔다.
- 모바일 우선 고려: 쓰기 인터랙션은 터치 환경을 기본 기준으로 설계한다.
- 단어 연결: 자모 학습은 예시 단어와 함께 제공한다.

## 7. MVP 범위

### 포함

- 기본 자음 14개, 기본 모음 10개
- 자모별 획순 애니메이션
- 획 단계 분해 보기
- 따라 쓰기용 워크북형 연습 화면
- 자모별 예시 단어 1~3개
- 모바일 반응형 UI
- 터치 기반 직접 쓰기 캔버스의 1차 버전

### 제외

- 이중모음, 겹자음, 받침 확장
- 사용자 계정 기반 진도 저장
- 정교한 필기 판정 알고리즘

## 8. 콘텐츠/데이터 원칙

### 획순 데이터

- 자모별 획수, 획 순서, 시작점, 진행 방향, 설명 문구를 관리한다.
- 교재 원본, 이미지형 획순 자료, 공신력 있는 참고 자료를 교차 검증한다.
- 자료 간 충돌 시, 이미지형 획순 자료와 내부 검증 메모를 우선한다.
- 현재 데이터는 확정본이 아니며 전수 검증이 필요하다.

### 예시 단어

- 자모별로 학습자 난도가 낮은 예시 단어를 붙인다.
- 단어는 가능하면 1~2음절 중심으로 시작한다.
- 예시 단어는 글자 모양 학습을 방해하지 않도록 1차 MVP에서는 소수만 제공한다.

## 9. UX 시나리오

1. 사용자는 홈에서 자음 또는 모음 학습을 선택한다.
2. 자모 상세 화면에서 획순 애니메이션을 본다.
3. 단계별 보기에서 각 획을 순서대로 확인한다.
4. 따라 쓰기 화면에서 가이드 위에 직접 따라 쓴다.
5. 직접 쓰기 화면에서 빈 칸에 한 번 더 써 본다.
6. 예시 단어를 보고 해당 자모가 들어간 단어를 함께 익힌다.

## 10. 화면 목록

### 10.1 홈

- 오늘 학습 시작
- 자음/모음 진입
- 최근 학습 자모
- 추천 학습 경로

### 10.2 자모 목록

- 자음/모음 탭
- 자모 카드 그리드
- 진행 상태 표시
- 검색 또는 빠른 이동은 후순위

### 10.3 자모 상세

- 큰 글자 미리보기
- 획순 애니메이션
- 획 단계 설명
- 예시 단어 카드
- 다음 자모 이동 버튼

### 10.4 따라 쓰기

- 흐린 가이드 글자
- 단계별 칸 연습
- 다시 쓰기
- 이전/다음 단계 이동

### 10.5 직접 쓰기

- 빈 칸 캔버스
- 획순 힌트 보기/숨기기
- 전체 지우기
- 한 획 되돌리기

### 10.6 퀴즈 또는 확인

- 올바른 획순 고르기
- 다음 획 맞히기
- 예시 단어와 자모 연결하기

## 11. 모바일 고려 사항

쓰기 인터랙션을 넣으려면 모바일 고려는 필수다. 이 프로젝트의 직접 쓰기 경험은 데스크톱보다 모바일 터치 환경에서 더 자주 사용될 가능성이 높다.

### 모바일 기준 설계 원칙

- 기본 입력은 마우스보다 터치를 우선한다.
- 버튼과 탭 영역은 한 손 조작이 가능한 크기로 설계한다.
- 쓰기 캔버스는 세로 화면 기준으로 우선 설계한다.
- 손가락으로 가려지는 문제를 줄이기 위해 현재 획 가이드는 캔버스 바깥에도 함께 보여 준다.
- 스크롤과 드로잉이 충돌하지 않도록 캔버스 영역의 터치 동작을 분리한다.

### 쓰기 인터랙션 요구사항

- `pointer events` 기반 입력 처리
- 손가락/펜 입력 모두 지원
- 최소한의 지우기, 다시 쓰기, 획순 힌트 제공
- 저사양 모바일에서도 부드럽게 동작하는 경량 구현

## 12. UI 개선 방향

- 현재 UI는 프로토타입 성격이 강해 정보 우선순위가 약하다.
- 학습 중심 화면으로 구조를 더 단순하게 재정리해야 한다.
- 획순, 설명, 따라 쓰기, 예시 단어가 한 화면에서 서로 경쟁하지 않게 해야 한다.

### 개선 포인트

- 상단 히어로 영역 축소
- 자모 선택 패널의 밀도 조정
- 획순 캔버스 가독성 개선
- 단계 설명 카드 간결화
- 예시 단어 카드 추가
- 모바일 레이아웃 재설계

## 13. 성공 기준

### MVP 완료 기준

- 자모 획순 데이터 24개 전수 검증 완료
- 자모별 예시 단어 제공
- 모바일에서 직접 쓰기 기본 동작 가능
- 한 글자 학습 흐름이 끊기지 않음
- 사용자가 `보기 → 따라 쓰기 → 직접 쓰기`를 1분 내 수행 가능

## 14. 주요 리스크

- 자료마다 획순 표기가 다를 수 있어 데이터 확정이 늦어질 수 있다.
- 모바일 드로잉 UX가 어설프면 학습 경험이 오히려 나빠질 수 있다.
- 예시 단어 선정이 난도나 문화 맥락 측면에서 부적절할 수 있다.

## 15. 다음 단계

- 획순 데이터 검증 루틴을 먼저 확정한다.
- 예시 단어 데이터 구조를 설계한다.
- 모바일 쓰기 캔버스의 기술 방향을 정한다.
- UI를 학습 흐름 중심으로 개편한다.
