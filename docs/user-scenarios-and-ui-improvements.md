# JamoFlow User Scenarios And UI Improvements

## 목적

사용자 시나리오를 학습 흐름 기준으로 정리하고,
각 시나리오에 맞는 UI 개선 스텝을 연결한다.

이 문서는 `Pipeline B 사용자 시나리오 개발`과
`Pipeline C UI 개선 정리`를 한 묶음으로 운영하기 위한 기준 문서다.

## 연결 워크플로우

```text
사용자 시나리오 정의
  -> 화면별 행동 흐름 정의
  -> UI 병목 파악
  -> 개선 우선순위 결정
  -> 구현/검토 반영
```

## 시나리오 1. 처음 온 입문 학습자

### 상황

- 사용자는 한글 자모를 처음 배우기 시작한다.
- 빠르게 첫 학습에 들어가고 싶다.

### 목표

- 홈에서 바로 학습 시작
- 자음/모음 중 하나를 선택
- 첫 자모 상세로 자연스럽게 이동

### 현재 병목

- 홈이 아직 실제 제품형 진입 화면으로 분리되지 않음
- 학습 경로 안내가 약함

### UI 개선 스텝

1. 홈에 `모음부터 시작`, `자음부터 시작` CTA 고정
2. 최근 본 자모 대신 `오늘의 시작 자모`를 우선 노출
3. 첫 진입 시 상세 화면으로 바로 이동하는 흐름 설계

## 시나리오 2. 특정 자모만 복습하려는 학습자

### 상황

- 사용자는 이미 몇 개의 자모를 배웠다.
- 특정 자모만 다시 보고 싶다.

### 목표

- 자모 목록에서 빠르게 자모 선택
- 현재 검증 상태를 같이 확인
- 상세 화면에서 획순과 예시 단어를 다시 본다

### 현재 병목

- 자모 목록의 정보 밀도는 아직 중간 수준
- `완료/1차/재검토`는 보이지만 학습 우선순위 연결은 약함

### UI 개선 스텝

1. 자모 목록 카드 크기를 조금 줄여 한 화면 정보량 증가
2. `재검토` 자모는 시각적으로 더 분명히 표시
3. 자모 선택 후 스크롤 위치와 포커스 유지

## 시나리오 3. 자모 하나를 끝까지 학습하려는 학습자

### 상황

- 사용자는 한 자모를 보고, 따라 쓰고, 직접 쓰기까지 한 번에 하고 싶다.

### 목표

- 상세 화면에서 획순 이해
- 따라 쓰기에서 단계 연습
- 직접 쓰기에서 빈 칸 연습

### 현재 병목

- 상세, 따라 쓰기, 직접 쓰기 간 흐름 연결이 아직 약함
- 단계 전환 CTA가 부족함

### UI 개선 스텝

1. 상세 화면 하단에 `따라 쓰기 시작` CTA 추가
2. 따라 쓰기 아래에 `직접 써 보기` CTA 추가
3. 직접 쓰기 완료 후 `다음 자모` 또는 `다시 연습` CTA 추가

## 시나리오 4. 모바일에서 손가락으로 연습하는 학습자

### 상황

- 사용자는 휴대폰에서 손가락으로 자모를 써 본다.

### 목표

- 캔버스가 충분히 크다
- 손가락으로 써도 스크롤과 충돌하지 않는다
- 힌트, 지우기, 다시 쓰기가 쉽게 눌린다

### 현재 병목

- 직접 쓰기 프로토타입은 들어갔지만
  하단 액션 구조와 다음 단계 이동은 아직 약함

### UI 개선 스텝

1. 모바일에서 캔버스 높이 우선 확보
2. 버튼 묶음을 하단 고정 또는 하단 근처 유지
3. 힌트/지우기/다시 쓰기 우선순위 정리
4. 완료 후 다음 단계 안내 추가

## 시나리오 5. 예시 단어로 기억하려는 학습자

### 상황

- 사용자는 획순만으로는 잘 기억되지 않는다.
- 단어와 연결해서 기억하고 싶다.

### 목표

- 자모 상세에서 단어를 바로 본다
- 너무 어렵지 않은 단어를 만난다
- 같은 자모가 들어간 단어를 함께 비교한다

### 현재 병목

- 예시 단어 카드는 들어갔지만
  학습 흐름에서 역할이 아직 약하다

### UI 개선 스텝

1. 대표 단어와 보조 단어의 시각적 위계 구분
2. 단어 카드에 target jamo 강조 표시 추가 검토
3. 추후 퀴즈 화면과 연결 가능한 구조 유지

## 화면별 UI 개선 우선순위

### 1순위

- 자모 상세
- 직접 쓰기

### 2순위

- 자모 목록
- 따라 쓰기

### 3순위

- 홈
- 확인/퀴즈

## 다음 구현 순서

1. 상세 -> 따라 쓰기 -> 직접 쓰기 CTA 연결
2. 모바일 직접 쓰기 하단 액션 구조 개선
3. 자모 목록 카드 밀도 개선
4. 홈 진입 화면 분리

## 현재 진행 상태

- 홈 진입 흐름 반영 완료
- 직접 쓰기 최소 피드백 반영 완료
- 자모 상세 우선순위 미세 조정 반영 완료
